## 📋 项目概述

这是一套完整、专业的“智能在线学习平台”软件需求分析文档，是在《软件需求工程》课程中完成的综合实践项目。项目严格遵循软件工程标准流程，从零开始对一款面向在校学生、在职人员、教育机构的智能学习平台进行了全面、系统的需求分析、建模和规格说明。

项目亮点:
* 全流程需求工程实践：完整覆盖了从业务需求分析 → 涉众需求获取 → 需求建模 → 规格说明的标准化流程
* 多维度建模能力：运用UML的用例图、类图、顺序图、活动图、数据流图、E-R图等多种模型，从不同视角刻画系统
* 工业级文档规范：采用企业标准的文档模板和格式，产出可直接用于实际开发的需求基线
* 深度需求挖掘：通过结构化的涉众访谈和问题分析，识别并解决了三大用户群体的核心痛点

## 📁 文档结构与内容
1. 项目前景和范围 (项目前景和范围.docx)
核心内容：定义项目的商业背景、目标、范围边界和产品定位。
关键产出：
* 业务目标分析：明确9个核心业务目标（学习进度跟踪、智能推荐、学习评估等）
* 产品定位声明：清晰定义目标用户、核心价值、与竞品（中国大学MOOC、学习通）的差异化
* 范围管理：制定三版本发布计划，明确各版本功能边界
* 涉众分析：识别5类关键涉众（学生、在职人员、教育机构、技术团队、运营团队）及其诉求


2. 涉众请求 (涉众请求.docx)
核心内容：通过结构化访谈，深入收集和分析三类核心用户（学生、在职人员、教育机构）的真实需求与痛点。
关键产出：
* 深度问题分析：对每类用户提出6-8个关键问题，分析“为什么存在→当前如何解决→期望如何解决”
* 需求优先级矩阵：将散乱的用户问题归类、整合，映射到具体的解决方案并划分优先级 
* 解决方案评估：在关键方案（如“线上vs线下学习”、“文字vs视频答疑”）上，基于评估准则进行理性决策

* 需求结构化建模 (需求结构化建模.docx)
核心内容：使用结构化方法对系统进行逻辑建模，明确系统与外部实体的交互、内部数据处理逻辑和核心数据结构。
关键产出：
* 上下文图：界定系统边界与外部实体
* 分层数据流图：描绘“用户登录”、“核心功能”、“课程管理”、“社区交流”四大关键流程的数据流转
* E-R图：设计包含用户、课程、教育机构、学习记录等9个实体的概念数据模型
* 数据字典：为E-R图中每个实体/属性明确定义，包含中文描述、数据类型、约束、示例，可直接用于指导数据库设计


4. 分析模型 (分析模型.docx)
核心内容：使用面向对象方法对系统进行动态和静态建模，聚焦系统内部对象间的协作。
关键产出：
* 用例模型：识别管理员、教育机构、用户等角色，定义用户注册、学习路径规划、课程推荐等核心用例，明确包含(include)/扩展(extend)关系
* 类图：定义用户类、课程类、学习记录类等核心类及其间的关联、依赖关系
* 顺序图：详述用户注册、学习评估、个性化推荐等关键场景中，对象间按时间顺序的消息交互
* 活动图：展示与顺序图对应的业务活动流程，特别突出循环、判断等逻辑

5. 需求规格说明 (需求规格说明.docx)

核心内容：将前述所有分析成果，转化为明确、可验证、可供开发和测试团队使用的需求规格。
关键产出：
* 对外接口需求：明确用户界面风格、硬件/软件/通信接口标准
* 详细功能需求：用结构化模板(说明-角色-输入-过程-输出-规则)定义用户注册、课程搜索、学习评估等6个核心功能



## 🔍 如何查阅这些文档

本目录下的每个 .docx文件对应上述一份独立文档。

建议按 1→2→3→4→5 的顺序阅读，以遵循从宏观到微观、从业务到技术的逻辑。
